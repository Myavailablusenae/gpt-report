01 —

從 Transformer 到 GPT

2017 年，Attention Is All You Need 論文的問世，讓 Transformer 架構迅速分化出三條演化路徑：以 BERT 為代表的編碼器（Encoder-Only）陣營、以 T5 為代表的混合（Encoder-Decoder）陣營，以及本報告的主角——以 GPT 為首的純解碼器（Decoder-Only）架構。

📌 核心概念： GPT 採取「自回歸（Autoregressive）」路徑——僅透過預測下一個 Token 的簡單目標，在百億級參數的推升下，展現出令人驚訝的零樣本學習（Zero-shot Learning）與能力湧現。

🤔 現代 LLM 為何幾乎全面倒向解碼器架構？

  • 規模法則（Scaling Laws）： 解碼器架構的效能隨參數、算力呈冪律增長，完美契合現代 GPU 並行算力。
  • 推理湧現： In-Context Learning 與思維鏈（CoT）完全奠基於自回歸生成框架。
  • ⚡ 計算效率： 解碼器在生成時可快取 $K$、$V$ 矩陣（KV Cache），推論速度遠快於編碼器-解碼器混合架構。
💡 湧現案例：GPT-3（175B 參數）首次展現「上下文學習」——只需 2-3 個範例就能完成從未訓練過的任務，而 GPT-2（1.5B）完全無法做到。
📌 批判性反思： 純解碼器架構擅長生成，但在「理解全文語境再回答」的任務（如閱讀理解、情感分類）上，表現仍不如雙向的 BERT。實務上會依任務混用不同架構。
02 —

技術核心：注意力機制與遮罩

GPT 的核心能力建立在 Scaled Dot-Product Attention（縮放點積注意力） 機制上。模型透過將輸入映射至 $Q$（Query）、$K$（Key）、$V$（Value）矩陣進行動態權重計算。

Scaled Dot-Product Attention 的五步核心
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

核心計算層

步驟 1

線性投影：得到 Q、K、V

輸入向量經三個獨立權重矩陣變換，產生 Query（查詢）、Key（鍵）、Value（值）三個表示。

步驟 2

點積相似度：計算 $QK^T$

Query 與 Key 的點積得到「關聯分數矩陣」，數值越大代表語義相關性越強。

步驟 3

梯度穩定：除以 $\sqrt{d_k}$

方差縮放防止高維點積過大，穩定 softmax 後的梯度，避免訓練時梯度消失。

正則化與融合層

步驟 4

機率正則化：$\text{softmax}(\cdot)$

將分數轉換為總和為 1 的機率分布（注意力權重），形成「注意力地圖」。

步驟 5

加權融合：乘上 V

注意力權重對 Value 向量進行加權求和，輸出包含上下文資訊的新向量表示。

GPT 獨有機制

GPT 獨有機制： 在步驟 2 計算 $QK^T$ 前，加入因果遮罩（Causal Mask），強制未來位置的分數設為 $-\infty$，經 softmax 後權重歸零，確保生成時遵循時序。

核心洞察： 注意力機制的精妙在於用「軟選擇」（softmax 權重）取代「硬選擇」（one-hot），讓每個位置都能與所有上文建立加權連接。這種動態、可微分的資訊路由方式，是 Transformer 相比 RNN 的根本優勢。

縮放因子 $\sqrt{d_k}$ 的物理意義： 當特徵維度 $d_k$ 極大時，加入 $\sqrt{d_k}$ 進行方差縮放，能防止 $\text{softmax}$ 後的梯度陷入極平緩區域，避免反向傳播時的梯度消失。

03 —

參數即智慧：GPT 演化史與範式轉移

隨著參數與訓練方法的演進，模型經歷了五次核心範式的轉移， 這不僅是算力與資料規模的暴力突破，更是人工智慧從 「特定任務工具」演進為「自主行動代理人（Autonomous Agent）」的實踐：

2018

基礎驗證期：無監督預訓練的潛力 (GPT-1 / GPT-2)

原始論文 "Language Models are Unsupervised Multitask Learners" 證實了單純透過海量文本進行「接龍預測」，模型就能在不修改網絡結構的 情況下，具備零樣本學習（Zero-shot）的泛化可能。 這階段奠定了「大數據預訓練 ➔ 少量下游微調」的技術路線。

無監督預訓練 Zero-shot 雛形 1.5B 參數上限
2020

規模湧現期：Scaling Law 與提示工程 (GPT-3)

參數暴力提升至 1,750 億。論文 "Language Models are Few-Shot Learners" 揭示了當模型規模跨越臨界點時，會「湧現（Emergence）」出未曾被刻意訓練的 推理能力。此時模型不需要修改任何權重，僅透過給予範例（Few-shot）的 上下文學習（In-context Learning），即可適應全新任務， 進而誕生了「提示工程（Prompt Engineering）」這一全新人機界面。

Scaling Law 實證 能力湧現 In-context Learning
2022

行為對齊期：指令微調與安全對齊 (ChatGPT / GPT-4)

這是模型從「純統計機率接龍」轉化為「可信賴助理」的關鍵。 透過有監督微調（SFT）與 人類回饋強化學習（RLHF），將 AI 的輸出行為與 人類的價值觀安全對齊（Alignment），使其具備強大的 思維鏈（Chain-of-Thought, CoT）推理能力，並為後續發展資安防禦 或企業決策應用提供了可防護的底座。

RLHF 安全對齊 思維鏈 (CoT) ChatGPT 破億用戶
2024

多模態融合期：感知擴展與推理模型誕生 (GPT-4o / o1)

GPT-4o（omni）整合文字、圖像、音訊與影片為單一模型， 真正實現跨感知通道的即時理解。同年底，o1 的問世代表 另一條新的技術路線正式登場：模型在回答前會先進行 內部推理步驟（Extended Thinking）， 以逐步推演取代直覺生成，在數學、程式、複雜邏輯上 大幅超越傳統 GPT 系列。

多模態融合（Omni） 推理前思考 o-series 誕生
2025

自主工具使用期：GPT × o-series 大融合 (o3 / o4-mini / GPT-5)

o3 在針對一般智能設計的 ARC-AGI 基準測試中首次超越人類平均， 並開創了「主動工具調用（Agentic Tool Use）」—— 推理模型能自主決定何時使用網路搜尋、程式執行、圖像分析等工具。 GPT-5（2025 年 8 月）正式合併兩條路線：快速通用的 GPT 系列 與深思熟慮的 o 系列統一為單一架構，成為兼具快速對話與 深度推理的 AI Agent 推理大腦。

o3 突破 ARC-AGI GPT × o-series 統一架構 原生 Agentic 能力
2026

自主行動期：電腦操作與快速迭代 (GPT-5.x)

GPT-5.4 帶著原生電腦使用能力（Computer Use）問世， 模型從「螢幕中的對話框」正式演變為「能操作軟體的 Agent」。 OpenAI 進入每月迭代週期（GPT-5.1 → 5.2 → 5.3 → 5.4 → 5.5）， 幻覺比前代減少六倍，並能在橫跨法律、工程、物流、銷售等 40 種職業的任務中，與領域專家並駕齊驅甚至超越。

Computer Use 月度快速迭代 Agent 正式商業化

04 —

資安實務（OWASP LLM Top 10）

將 GPT 導入企業環境（如 IDE 輔助或網路監控），必須正視大型語言模型特有的風險：

LLM01: 提示注入

惡意輸入覆蓋原始指令，繞過系統安全護欄。

LLM03: 訓練資料汙染

微調資料遭竄改，導致模型輸出隱蔽的惡意後門。

LLM06: 敏感資訊外洩

記憶體重現導致 PII 或開發 API Key 遭提取。

LLM09: 幻覺誤導

生成看似合理卻錯誤的資安告警，引發錯誤決策。

實務整合建議採用：內容安全過濾器、最小權限原則，並結合 RAG 架構降低幻覺發生率。

05 —

學習反思：幻覺的本質

AI 不是真理機器，而是基於海量文本的高階機率預測器。

透過本次查證，最深刻的體悟是：GPT 本質上是最大化「給定上文時，下個 token 的條件機率」。它生成的知識是機率分佈的產物，而非經驗證的事實。幻覺現象是統計模型對抗不確定性時的必然結果。

未來展望

深入理解「因果遮罩」與演算法底層，才能掌握 AI 邏輯，而非淪為單純的 API 呼叫者。期許自己未來能開發出兼具效能與安全對齊（Alignment）的 LLM 應用。
