一、從 Transformer 到 GPT：解碼器架構的崛起

在 2017 年 “Attention Is All You Need” 論文問世後，Transformer 架構分化出了三種不同的演化路徑：以 BERT 為代表的編碼器（Encoder-Only）陣營、以 T5 為代表的混合（Encoder-Decoder）陣營，以及本報告的主角——以 GPT 為首的純解碼器（Decoder-Only）陣營。

與傳統追求雙向語境理解的模型不同，GPT 採取了更具挑戰性的「自回歸（Autoregressive）」路徑，僅透過預測下一個 Token 的簡單目標，卻在百億級參數的推升下，展現出了令人驚訝的零樣本學習（Zero-shot Learning）與能力湧現。GPT 系列向世界證明了：足夠大的解碼器 + 足夠多的資料 + 自回歸預訓練，能夠讓語言模型產生「推理」的雛形。

  • 演算法定義： 基於 Transformer Decoder 的自回歸語言模型，每次生成一個 token 並以先前輸出為條件。
  • 核心解決痛點： 徹底屏棄傳統 RNN 難以並行訓練與無法有效處理長距離依賴（Long-range Dependency）的問題，同時透過因果遮罩實現合乎邏輯的序列生成。
  • 現代實務應用： 智慧程式碼輔助生成（如 GitHub Copilot）、自動化資安弱點分析報告、自主 AI Agent 推理核心、即時對話代理人。

相較於 BERT 擅長「理解」任務，GPT 架構的優勢在於「創造」與「續寫」。從 GPT-3 的提示學習到 ChatGPT 的 RLHF 對齊，再至 GPT-4 的多模態能力，純解碼器架構已成為生成式 AI 的事實標準。

二、技術前沿：自注意力機制與數學底層

GPT 的核心能力建立在 Scaled Dot-Product Attention（縮放點積注意力） 機制上。模型透過將輸入向量映射至三個獨立的矩陣：$Q$ (Query)、$K$ (Key)、$V$ (Value) 進行動態權重計算，其底層數學公式如下：

$$Attention(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

🔍 關鍵技術查證與修正（對應評量任務 3）

在查證原始文獻時，多數通俗教學常忽略分母中縮放因子 $\sqrt{d_k}$ 的物理意義。經深入論文查證發現：當特徵維度 $d_k$ 極大時，點積（Dot-Product）的數值會劇烈增長，導致經由 $\text{softmax}$ 函數計算後的梯度陷入極其平緩的區域。加入 $\sqrt{d_k}$ 能有效進行方差縮放，防止在反向傳播（Backpropagation）時發生梯度消失現象，穩定訓練過程。

此外，GPT 作為 Decoder-Only 模型，在進行自注意力計算時，會強制引入一個上三角的遮罩矩陣（Masking Matrix）。此機制使模型在預測第 $n$ 個 Token 時，其 Attention 權重無法分配給第 $n+1$ 之後的未來資訊，確保了其嚴格的自回歸特性。下圖展示了 4×4 的因果遮罩概念（灰色區塊代表遮蔽位置，模型無法看見未來 token）。

💡 智庫觀點：圖書館索引模擬（給非技術決策者）

給政商與非技術決策者的報告： 各位可以想像 GPT 內部運行著一座無限大的圖書館。當使用者給出指令時，它並非在僵硬地查閱字典，而是同步進行三件事：第一，確認你到底在問什麼（Query）；第二，掃描所有架上資料的標籤（Key）；第三，根據匹配程度，精準提取最有價值的內容（Value）。

這意味著大語言模型的強大不在於它「背下」了死板的知識，而在於它極其高效的資源動態調度能力。這也是為什麼我們不能將其當作百分之百正確的「知識百科」，而應將其視為一個具備極高產出效率的「機率預測與推理引擎」。同時，位置編碼與多頭注意力的協同作用使得 GPT 能掌握語序與層次語義。

多頭注意力（Multi-Head Attention）進一步將 $Q,K,V$ 投影到多個子空間，讓模型能同時關注不同範圍的上下文關係，這也是 GPT 能捕捉複雜語法結構與篇章邏輯的關鍵。

三、實務應用與安全性（OWASP Top 10 視角）

將 GPT 導入企業軟體開發、資安監控及對話系統時，除了效能增益，也必須正視大型語言模型特有的資安風險。OWASP 基金會於 2023 年發佈了 LLM Top 10 風險清單，以下摘錄與 GPT 實務最相關的威脅與對策：

🔓 提示注入
攻擊者透過精心建構的輸入覆蓋原始指令，可能導致模型洩漏系統提示或繞過安全護欄。
📊 訓練資料 poisoning
若預訓練或微調資料遭汙染，模型可能輸出後門或偏差內容，需嚴格的資料溯源。
⚖️ 敏感資訊外洩
模型可能記憶並重現訓練集中的機敏資料 (PII、API Key)，需搭配差分隱私與輸出過濾。
🎭 幻覺導致錯誤決策
生成看似合理但錯誤的資訊，在自動化報告或資安告警場景可能造成嚴重誤判。

實務上整合 GPT 時，可採用內容安全過濾器、最小權限原則、定期紅隊演練，以及將模型輸出與可靠知識庫（RAG）檢索結合以降低幻覺。本學期透過實務案例探討，認知到「AI 並非信任根源，而是可信輔助工具」。

應用亮點：利用 GPT 自動生成弱點掃描的解讀摘要、加速資安事件回應劇本撰寫，以及對抗社交工程郵件的分析。但務必搭配人機協作與稽核機制，方能實現安全增益。

四、學習反思：統計引擎與幻覺的本質

透過這個學期對原始論文《Attention Is All You Need》與 GPT 系列技術報告的查證與實作，最大的體悟是：AI 不是真理機器，而是基於海量文本的高階機率預測器。GPT 本質上是在最大化「給定上文時，下一個 token 的條件機率」，因此它所生成的「知識」其實是機率分佈下最合理的組合，而非經過驗證的事實。

所謂「幻覺 (Hallucination)」現象，正是模型在低信心區域或缺乏訓練資料時，選擇了高機率卻錯誤的路徑。這不代表模型「說謊」，而是反映了統計模型在對抗不確定性時的本質缺陷。因此在專業領域（醫療、法律、關鍵基礎設施）採用 GPT 時，必須引入外部驗證鏈與檢索增強生成（RAG）。

📌 我的觀點與未來方向

本次報告亦讓我反思：未來開發者不應迷信模型規模盲目擴大，反而應追求可解釋性注意力與輕量化推論架構。同時，理解「因果遮罩」與「位置編碼」等細節，才能真正掌握 GPT 背後的運算邏輯，而非停留在 API 呼叫層次。透過這份期末報告，我深刻理解為何 Transformer 被譽為「改變深度學習格局」的里程碑，也期許自己能投入更具安全意識的生成式 AI 應用開發。

總結來說，GPT 演算法賦予了機器驚人的語言流暢度，但作為新一代資訊人員，我們必須同時具備演算法素養與風險意識，才能在 AI 浪潮中穩健前進。
